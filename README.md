기존의 머신 러닝은 통계 기반이기 떄문에 데이터를 하나씩 처리해야했음 (CPU) *비동기처리로 하면 속도 개선

### 1) 인공지능 (AI)
- 컴퓨터가 인간과 같은 지능적인 행동을 할 수 있게 해주는 기법들을 연구하는 분야
- EX) 이전 게임의 NPC : 심볼릭 AI(if 문의 복잡한 구성)

### 2) 머신러닝
- 인공지능 연구분야의 방법론 중 하나로 명시적인 프로그래밍 없이 컴퓨터가 학습하는 능력을 갖추게 하는 분야
- 데이터를 이요해 컴퓨터가 지식이나 패턴을 학습/추론 하는 것
- 지도학습 ,비지도학습, 강화학습 등의 알고리즘이 있음
- sklearn 이 대표적인 패키지


### 3) 딥러닝
- 지도학습의 여러 방법론 중 하나
- 인공신경망을 깊게 여러 층 쌓아 올려 학습하는 방식
- 문자열 (자연어) , 이미지, 영상처리 등을 위해 사용
- Tensorflow 가 대표적인 패키지 예시


## #02. 머신러닝
### 머신러닝 알고리즘 대분류
<회귀분석>
- 과거의 데이터가 충분한지?
- 데이터의 양이 충분한지?
- 예측하고자 하는 종속 변수가 있는가?
- 종속변수가 수치형인가?

<분류분석>
- 과거의 데이터가 충분한지?
- 데이터의 양이 충분한지?
- 예측하고자 하는 종속 변수가 있는가?
- 종속변수가 범주형인가?

<군집분석>
- 과거의 데이터가 충분한지?
- 데이터의 양이 충분한지?
- 유사한 관측치를 묶고자 하는가?
  (종속변수가 없을 떄는 군집분석)

### 학습 유형에 따른 알고리즘 분류

#### 지도학습
- 의도하는 결과(정답) 이 있을 때, 즉 종속 변수가 존재할 때 사용
- 이미 알려진 사례를 바탕으로 일반화된 모델을 만들어 의사 결정 프로세스 자동화
- 알고리즘에 입력값, 기대 출력값 제공 -> 알고리즘은 주어진 입력에서 원하는 출력을 만드는 것
- 크게 분류와 회귀 예측으로 나누어짐
  
|구분|예시|
|---|---|
|분류|스팸 필터, 질병 판별|
|회귀|주행거리,브랜드 특성으로 중고차 가격 예측|

> 분류에서 종속변수는 label , 회귀에서는 target 이라고 부름
> 독립변수는 feature 이라 부름

- 지도학습을 여러개 겹쳐 쓰면 앙상블 기법 EX) (분류) decision tree -> random forest -> XGBoost
  

#### 비지도 학습
- 훈련 데이터에 의도하는 결과(레이블)이 없을 때 사용
- 입력 데이터를 비슷한 유형의 집합으로 분류
- 즉, 컴퓨터가 데이터의 특성을 스스로 파악해서 분류하는 것 (사람이 주는 입력값 없음)


- ** 블로그 글 주제 구분**
   - 텍스트 요약, 핵심 주제 찾기
   - 사전에 어떤 주제인지 알지 못함
   - 하여, 출력값을 준비할 수 없다

- 고객들을 위향을 기준으로 묶기
   - 고객 데이터로 취향을 그룹핑 하고자 함
   - 어떤 그룹이 있는지 미리 알 수 없어 출력을 가지고 있지 않음

- 비정상적인 웹사이트 접근 탐지
   - 일상적이지 않은 패턴 탐지
  
#### 강화학습
- 결정을 내릴 떄 사용
- 결정을 했을 때 성공/실패에 따라 주어진 입력값에 대한 결정이 달라짐
- 학습을할 수록 입력에 대한 결과 예측이 가능해짐
- 보상의 개념 필요 ex) 포인트-> 게임 NPC
- 일반적인 데이터 분석에서는 활용도가 낮음
